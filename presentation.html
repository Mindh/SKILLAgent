🧠 AI 모델(LLM)의 역할

LLM은 Large Language Model의 약자 — 한마디로 "매우 똑똑한 자동완성"입니다

👨‍💼

일반 직원이라면

  • • 사수에게 "이걸 어떻게 해요?" 물어본다
  • • 매뉴얼 보고 판단한다
  • • 계산기 꺼내서 숫자를 계산한다
  • • 사내 시스템에서 직원 정보를 찾는다
≈
🤖

AI 모델이라면

  • • 시스템 프롬프트(지시사항)를 읽는다
  • • 대화 맥락을 파악해서 판단한다
  • • calculator 도구를 호출한다
  • • employee_lookup 도구를 호출한다
핵심 역할: AI는 스스로 계산하거나 DB를 뒤지지 않습니다. "지금 무엇을 해야 할지 결정"하고, 실제 작업은 도구에 맡깁니다.

📜 대화 히스토리 = 시스템의 기억

이 시스템은 별도의 "세션 저장소"가 없습니다. 대화 내용 자체가 모든 상태입니다

user [사용 가능한 도구 목록] calculator, employee_lookup, translate... (최초 1회)
assistant 네, 사용 가능한 도구 목록을 확인했습니다.
user [워크플로우 컨텍스트 로드: leave_intake] — 휴직 접수 절차 전문...
assistant 네, 해당 워크플로우 절차를 참고하겠습니다.
user 육아휴직 신청하고 싶어요
assistant 안녕하세요! 육아휴직 신청을 도와드리겠습니다. 먼저 신청자 이름과 사유를 알려주시겠어요?
🔑 AI는 매 응답마다 이 전체 대화 목록을 처음부터 읽습니다. 따라서 앞서 말한 내용을 기억하고, 워크플로우를 중간에 바꿨다가 돌아와도 맥락이 유지됩니다.

# 시스템이 판단하는 방식
도구 호출 없는 응답 → 최종 답변으로 사용자에게 전달
도구 호출 있는 응답 → 실행 후 결과 받아서 재응답
워크플로우 키워드 감지 → 해당 절차 주입 → 단계 안내

➕ 새 기능 추가는 얼마나 쉬운가요?

시스템은 파일만 추가하면 바로 확장됩니다. 기존 코드를 수정할 필요가 없습니다
